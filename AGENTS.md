# 家系図生成ツール（YAML to Graphviz）設計仕様書

## 1. 概要
YAML形式の定義ファイルから、Graphvizを利用して「世代（レイヤー）」の揃った、写真・属性付きの家系図（SVG）を自動生成する。

## 2. 実行環境・ライブラリ
- **Language:** Python 3.10+
- **Libraries:** - `PyYAML`: 設定ファイルの読み込み
    - `graphviz`: DOT言語の生成およびSVGレンダリング
- **External Tool:** `Graphviz` 本体のインストールが必要
- uvで管理

## 3. 入力データ仕様 (YAML)
以下の構造を持つYAMLファイルを読み込む。

```yaml
people:
  - name: "佐藤 太郎"
    reading: "さとう たろう"
    sex: male
    birthday: "1945-03-10"
    deathday: null # 存命の場合はnullまたは省略
    job: "農業"
    image_path: "photos/taro.jpg"
    parents: []
```

サンプルとして、サザエさん一家を定義した設定ファイルを作成して進める。

## 4. 描画・レイアウトロジック

### A. 世代（Rank）の制御
- **計算:** `parents`が空の人物をLevel 0とし、子供は `max(親のLevel) + 1` で計算する。
- **配置:** `graphviz.Digraph` の `subgraph(rank='same')` を使用し、同じLevelの人物を水平に一列に配置する。

### B. 配置の優先順位
1. **夫婦の左右:** 同じランク内で `sex: male` を左、`sex: female` を右に配置。
2. **兄弟の左右:** `birthday` を比較し、年長者を左に配置。
3. **夫婦の結合:** 夫と妻の間に不可視のポイント（Node）を作成し、そこから子供へエッジを引くことで、家系図特有の「夫婦からの分岐」を表現する。

### C. ノードのデザイン (HTML-like Label)
各人物のノードは `shape=plaintext` とし、内部をHTMLテーブルで構築する。
- **必須表示項目:**
    - 写真（`image_path`が空の場合は「No Image」のプレースホルダー）
    - ふりがな（フォントサイズ小）
    - 名前（太字）
    - 生年月日・計算された年齢（実行日時点）
- **年齢計算:** スクリプト実行日を基準日として算出する。
- **スタイル:** - `deathday` がある（故人）場合は、テーブルの境界線を「破線(dashed)」にする。
    - 性別によって背景色を微調整する（例：薄い青/薄い赤）。

### D. データの不備対応
- **生年月日なし:** 年齢表示を「不明」または非表示とする。
- **名前不明:** YAMLに `name: "不明"` と記述されている場合、一意のID（例：`unknown_1`）を内部で割り当て、図中に「不明」ノードを確保する。

## 5. 特記事項
- ID管理は `name` をキーとするが、同姓同名を許容する場合は内部的に `name + birthday` のハッシュ値をIDとして扱うこと。
- ノード内の画像は `IMG` タグを使用するため、Graphvizが参照できる相対パスを維持すること。
